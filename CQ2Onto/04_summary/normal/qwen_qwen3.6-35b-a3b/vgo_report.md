## Layer 3, Strict Alignment

For the **10** property pairs that the upstream alignment step matched (from `property_best_matching.csv`), each pair's domain and range are evaluated. ObjectProperty domain and range are compared via the class alignment table (`class_best_matching.csv`); DatatypeProperty range is compared via normalized XSD-type equality.

**Classification definitions** (per side, i.e. domain or range):

- `skip`, both gold and pred declared **nothing** on this side
- `TP`, both declared and they agree under the alignment
- `FP`, pred declared something but gold did not
- `FN`, gold declared something but pred did not
- `mismatch`, both declared but they disagree (counts as **both** FP and FN)

**Metric formula** (mismatch is double-counted):

```
FP_total = fp + mismatch       # pred-side errors
FN_total = fn + mismatch       # gold-side gaps
P = TP / (TP + FP_total)
R = TP / (TP + FN_total)
```

## Layer 3 — Axiom Alignment Check
For each gold axiom, look up its subject in the class or property alignment table. If pred has the same kind of axiom on the corresponding pred subject, compare the object from description logic representation: class names go through the class alignment table, property names through the property alignment table, datatypes by normalized string equality. Complex expressions (Intersection, Union, ∃p.C, ∀p.C, cardinality, OneOf, …) are broken down and each part is compared the same way.

**Classification definitions** (per gold axiom):

- `TP` term is aligned AND pred has a matching axiom
- `FN` term is aligned BUT pred has no axiom of this type for the corresponding pred term
- `mismatch` term is aligned AND pred has an axiom of this type, but the rest of the axiom does not align (counts as **both** FP and FN)
- `skip` gold term is NOT in the alignment table, this axiom is excluded from Layer 3 metrics (Layer 4 picks these up)

After scanning gold axioms, we scan pred axioms whose subject IS in the alignment table. If such a pred axiom wasn't matched to any gold axiom in the first pass, it counts as a **pure FP** in Layer 3 pred produced extra material on a term that gold also talks about. Pred axioms whose subject is NOT in the alignment table are not counted in Layer 3 at all; they're picked up by Layer 4 as `FP_unaligned`.

Each pred axiom is matched to **at most one** gold axiom (strict 1-to-1). Once a pred axiom is consumed as TP or Mismatch, it cannot be reused, preventing TP inflation when several gold axioms share the same subject.

**Metric formula** (mismatch is double-counted):

```
FP_total = fp + mismatch       # pred-side errors
FN_total = fn + mismatch       # gold-side gaps
P = TP / (TP + FP_total)
R = TP / (TP + FN_total)
```
