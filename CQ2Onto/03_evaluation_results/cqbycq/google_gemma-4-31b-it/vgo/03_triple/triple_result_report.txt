LAYER 3 — Strict Alignment (via best-matching tables)
────────────────────────────────────────────────────────────────────────
  10 property pairs from property_best_matching.csv.
  Domain evaluated via class_best_matching.csv;
  Range evaluated via class_best_matching.csv (ObjectProperty)
  or normalized XSD equality (DatatypeProperty).

  Evaluation criteria:
    skip      — gold None and pred None (nothing declared on either side)
    TP        — both declared and set-equal under the alignment table
    FP        — pred declared but gold did not
    FN        — gold declared but pred did not
    mismatch  — both declared but they disagree under the alignment table
                (mismatch contributes to both FP and FN)

  Metric formula (mismatch counts as both FP and FN):
    FP_total = fp + mismatch       (pred-side errors)
    FN_total = fn + mismatch       (gold-side gaps)
    P  = TP / (TP + FP_total)
    R  = TP / (TP + FN_total)
