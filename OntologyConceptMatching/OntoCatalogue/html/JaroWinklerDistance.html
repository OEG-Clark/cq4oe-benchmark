JaroWinklerDistance: String similarity metric emphasizing common prefixes

This metric aims to evaluate the similarity between generated label and the labels in the gold standard ontology with an emphasize of common prefixes.

Calculation Process:

  1. Set the matching window size based on the string lengths, which is the length of the longer string, divided by 2, round down and minus 1.

  2. Find all matching characters in the generated label and the labels in the gold standard ontology.

  3. Write down the matched characters in the order they appear in each string (using the matched positions). Compare these two matched-character sequences and convert mismatch count into the transposition count.

  4. Calcuate Jaro distances as: avg(sum(len(matched_character)/len(generated_label), len(matched_character)/len(gold_standard_label), (len(matched_character)-transposition_count)/len(matched_character)))

  5. Apply Winkler prefix boost: Jaro_distances + len(common_prefix)p(1 - Jaro_distances); noted p is constant scaling factor that is greater than 0 and smalled than 0.25.

  6. The range of jaro_winkler distance is from 0 to 1. 0 as no similarity shared between labels, and 1 means the identical labels.

Example:

Generate_Label: wine region

Gold_Standard_Label: wine type

Result:

  1. Windows Size: (max(len(Generate_Label), len(Gold_Standard_Label)) / 2) - 1 = 4

  2. Greedily find matching characters: for example Generate_Label[0] == Gold_Standard_Label[0]; Generate_Label[6] == Gold_Standard_Label[8], etc. Total matching character: 6

  3. Getting the transposition count: 0

  4. Jaro similarity result: (6/11 + 6/9 + (6-0)/6)/3 = 0.7374

  5. Winkler prefix boost: use the default setting of scaling factor, which is 0.1. therefore the jaro_winkler distance: 0.7374 + 40.1(1- 0.7374) = 0.8424

What is being measured?

This metric is particularly effective for labels sharing morphological roots or hierarchical naming patterns
